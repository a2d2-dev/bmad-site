什么是 BMAD？

BMAD 不是让 AI 替你思考，而是让 AI 作为专家协作者，引导你产出最佳想法

🎯

解决什么问题？

传统 AI 辅助开发容易产生「上下文丢失」和「偷懒输出」。BMAD 通过结构化工作流和专业 Agent，确保每个环节都有完整上下文。

🤖

核心理念

模拟真实开发团队的专业化分工：分析师调研、PM 写需求、架构师设计、开发者实现、QA 测试——每个角色都是专业 Agent。

📈

规模自适应

从修复一个 bug 到构建企业级系统，BMAD 自动调整规划深度。小项目快速执行，大项目深度规划。

🎭 代理化规划 (Agentic Planning)

传统 AI 生成的需求文档过于简单。BMAD 让专业 Agent 团队协作规划，产出企业级质量的 PRD 和架构文档。

  • • 专业代理团队协作
  • • 人机协作确保质量
  • • 详细输出，可执行落地

🧠 上下文工程 (Context Engineering)

开发过程中最大的问题是上下文丢失。BMAD 将完整上下文嵌入每个开发故事，开发者打开就知道做什么、怎么做。

  • • 智能故事生成
  • • 嵌入式知识
  • • 无缝交接

快速入门

3 步开始使用 BMAD

1

安装 BMAD

npx bmad-method install

运行安装器，选择需要的模块（默认安装核心模块 BMM）

2

打开 AI IDE

在项目文件夹中启动你喜欢的 AI IDE：

Claude Code Cursor Windsurf Cline
3

开始工作

/bmad-help

随时获取上下文感知的帮助，或直接召唤 Agent：

"请帮我做头脑风暴" → Mary 分析师上线

"请创建 PRD" → John PM 接管
