为什么选择 BMAD？

BMAD 不是另一个 AI 代码生成器。它是一个完整的 AI 驱动开发框架，通过结构化工作流和专业 Agent 帮助你构建软件。

🤔 传统 AI 工具的问题

  • ✗ AI 替你思考，产生"偷懒"输出
  • ✗ 开发过程中上下文丢失
  • ✗ 需求文档过于简单，缺乏深度
  • ✗ 没有结构化的开发流程

✨ BMAD 的解决方案

  • ✓ 专家协作者模式 — AI 引导而非替代
  • ✓ 上下文工程 — 每个故事包含完整信息
  • ✓ 代理化规划 — 多 Agent 协作产生企业级文档
  • ✓ 完整生命周期 — 从头脑风暴到部署

三大规划轨道

根据项目规模选择合适的规划深度

Quick Flow

快速轨道

1-15 Stories

适合：Bug 修复、简单功能、明确范围的小任务

产出： Tech-spec（技术规范）
流程： Tech-spec → 直接开发

BMad Method

标准轨道

推荐
10-50+ Stories

适合：产品、平台、复杂功能开发

产出： PRD + Architecture + UX
流程： PRD → 架构 → Epics & Stories → 开发

Enterprise

企业轨道

30+ Stories

适合：合规要求、多租户系统、企业级应用

产出： PRD + Architecture + Security + DevOps
流程： PRD → 架构 → 安全/运维 → 开发
