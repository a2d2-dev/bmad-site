什么是 BMAD？

BMAD (Build More Architect Dreams) 是一个 AI 驱动的敏捷开发框架， 通过专业化的 AI Agent 和结构化工作流，帮助开发者从想法到上线完成整个开发过程。

不同于传统的 AI 代码生成工具，BMAD 采用专家协作者模式： AI 不是替你思考，而是作为专家团队成员，引导你通过结构化流程产出最佳成果。

核心理念

1. AI 不是替你思考

传统 AI 工具会产生"偷懒"输出 — 简单的需求文档、缺乏深度的技术方案。 BMAD 认识到，AI 应该作为协作者而非替代者。

2. 上下文工程

开发过程中最大的问题是上下文丢失。BMAD 通过上下文工程， 将完整的项目背景嵌入每个开发 Story，确保 AI 始终有完整的上下文。

3. 规模自适应

从修复一个 bug 到构建企业级系统，BMAD 自动调整规划深度。 小项目快速执行，大项目深度规划。

两大核心创新

代理化规划 (Agentic Planning)

传统 AI 生成的需求文档过于简单。BMAD 通过专业 Agent 团队协作规划：

  • Analyst (Mary) — 市场分析和用户研究
  • PM (John) — 产品需求和项目管理
  • Architect (Winston) — 系统架构和技术设计

多个 Agent 协作，产出企业级质量的 PRD 和架构文档。

上下文工程开发 (Context Engineered Development)

Scrum Master 将规划转化为包含完整上下文的开发 Story：

  • 智能故事生成
  • 嵌入式知识 — 每个 Story 包含实现所需的所有信息
  • 无缝交接 — 开发者打开 Story 就知道要做什么、怎么做

专业 Agent 团队

BMAD 提供 12+ 专业 Agent，各司其职：

Analyst (Mary)

分析师 — 头脑风暴、研究

PM (John)

产品经理 — PRD、Epic

Architect (Winston)

架构师 — 架构设计

Scrum Master (Bob)

SM — Sprint 规划

Developer (Amelia)

开发者 — 代码实现

UX Designer (Sally)

UX 设计师

QA (Quinn)

QA 工程师 — 测试

模块生态

BMAD 采用模块化架构，可以扩展：

  • BMM — 核心框架，34+ 工作流
  • BMB — Builder，创建自定义 Agent 和工作流
  • TEA — Test Architect，测试策略和自动化
  • BMGD — Game Dev Studio，游戏开发
  • CIS — Creative Intelligence，创新和设计思维

Party Mode

Party Mode 是 BMAD 的独特功能 — 在单个会话中召唤多个 Agent 协作讨论。 就像真实团队会议一样，不同专家可以交换意见、达成共识。
